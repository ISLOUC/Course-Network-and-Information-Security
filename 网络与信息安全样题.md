#### 概念与术语  
【信息】（香农1948）任何可以减少不确定性的事物均为信息。  
【信息系统】（GB 17859）计算机信息系统是由计算机及其相关的和配套的设备、设施(含网络)构成的，按照一定的应用目标 和规则对信息进行采集、加工、存储、传输、检索等处理的人机系统。  
【信息资产】有价值的信息和信息系统，其价值可能是购买、维护信息系统的资金投入，或信息本身的经济价值，也可能是信息被窃取或破坏后对拥有者及其他人造成的损害，损害价值的评估既包括直接损失，也包括失去客户、投资者和名誉乃至法律责任的间接损失。  
【威胁】能够对信息资产产生破坏或滥用的潜在外部因素。既包括人为因素，也包括自然界和物理环境的因素。其中人为因素既包括来自合法用户的也包括来自非授权用户的，既包括故意的攻击也包括无意的误操作。  
【威胁主体】能够产生威胁的实体。  
【安全机制】用来对抗威胁、避免安全事件发生的技术和方法。  
【脆弱性】A weakness of an asset or group of assets which can be exploited by one or more threats.       
【风险评估/信息安全风险评估】风险管理国际标准ISO31000给出的定义：风险评估是风险识别、风险分析及风险评价的全过程。由此可见，信息安全风险评估是信息安全风险的识别、分析及评价的全过程。  
【风险】The potential that a given threat will exploit vulnerabilities of an asset or group of assets and thereby cause harm to the organization。  
【定性/定量风险评估】定性/定量风险评估是指在风险评估的风险分析阶段分别使用了定性的影响分析方法和定量的影响分析方法。其中，定量方法对风险分析过程中的所有要素进行定量赋值，通过等式得出系统的整体和残余风险。定性方法则不进行量化计算，而是通过主观性地分析、评价，给出等级化的风险分析结果。    
【社会工程】是通过欺骗，使他⼈泄露安全相关的敏感信息，或者按照欺骗者意图进⾏某些安全相关的操作。  
【主动攻击】对完整性和可⽤性的破坏。  
【被动攻击】对机密性的破坏。  
【机密性】信息不被未授权获取的性质。  
【完整性】信息不被未授权更改的性质。  
【数据完整性】数据不被未授权更改的性质。  
【系统完整性】系统的行为（如控制流）不受未授权更改的性质。  
【可用性】确保系统以足够的能力、预期的方式、可接受的性能提供服务。    
【拒绝服务攻击】是指攻击者通过某种手段，有意地造成计算机或网络不能正常运转从而不能向合法用户提供所需要的服务或者降低其提供的服务质量。  
【真实性】真实性即某主体的身份与其声称的身份相一致的特性。真实性通常通过身份鉴别的手段，即验证用户提交的某种与其声明的身份相绑定的信息的完整性来获得。但真实性与信源完整性有显著的区别：其一，真实性需要鲜活性，即重放的信息是无效的，这点区别于信源完整性；其二，某些具有信源完整性的消息是主体匿名的，这点又不符合真实性的要求。  
【认证/身份鉴别】通过用户提交的附加信息，对主体身份进行证实的过程。  
【非否认服务】防止参与某次通信交换的一方事后否认本次交换曾经发生过的服务称为非否认服务。  
【accountability（可追责性）】The traceability of actions performed on a system to a specific system entity (user, process, device). For example, the use of unique user identification and authentication supports accountability; the use of shared user IDs and passwords destroys accountability.（From Wiki）非否认性、完整性、日志记录是确保可追责性的基础。可追责性可支持安全审计。  
【审计】ISO/IEC 15408给出的定义：信息系统安全审计主要指对与安全有关的活动的相关信息进行识别、记录、存储和分析；审计记录的结果用于检查网络上发生了哪些与安全有关的活动，谁（哪个用户）对这个活动负责；主要功能包括：安全审计自动响应、安全审计数据生成、安全审计分析、安全审计浏览、安全审计事件选择、安全审计事件存储等。  
【攻击面（attack surface）】The attack surface of a software environment is the sum of the different points (the "attack vectors") where an unauthorized user (the "attacker") can try to enter data to or extract data from an environment.Keeping the attack surface as small as possible is a basic security measure.（来自wiki）  
【攻击树】用树型结构来描述系统所面临的安全威胁和系统可能受到的多种攻击，其中根节点代表被攻击的目标，叶节点表示达成攻击目标的方法。

【完善保密性】对于任意密文 y 和明文 x，都有 Pr[x | y] = Pr[x]。即，不能通过分析密文获得明文的任何信息。  
【信息熵】香农提出的用来量化信息的一种度量方式，用信源的不确定度来描述信源的信息含量。  
【无条件安全性】假定攻击者拥有无限的计算资源，但仍然无法破译该密码系统，则称该密码系统是无条件安全的。  
【计算安全性】使用目前最好的方法攻破密码系统所需要的计算量远远超出攻击者的计算资源水平，则这个密码体系为计算安全的。计算安全性具有时效性。     
【密码学】研究秘密通信为目的 包括：密码编码学、密码分析学。  
【明文】需要加密的消息。  
【密文】被加密后的消息。  
【加密算法】用于加密的数学函数。  
【可证明安全】由Goldwasser和Micali最先提出的方法，将密码系统的安全性归结（归约）为某个公认为求解困难的数学难题（如大整数素因子分解、计算离散对数等），或没有有效解法的问题（如NPC、NP-Hard问题等），实际上并没有证明算法是安全的，而是证明某个没有有效解法的问题不难于该密码系统的破解问题，即如果该密码系统被破解，则相对应的难题将找到有效解法。  
【非多项式有界量】函数f(n):N→R称为关于n的任意多项式无界的，IF对任意多项式p(n),存在一个自然数n0，使得对于任意n＞n0，都有f(n)＞p(n)。  
【可忽略量】函数ℇ(n):N→R称为关于n的一个可忽略量（ℇ(n)可忽略），IF其倒数1/ℇ(n)是关于n的一个非多项式有界的量。  
【压倒性概率】如果ℇ(n)是一个可忽略量，那么1-ℇ(n)就是一个压倒性的量/概率。  
【计算不可区分】two families of distributions are 【computationally indistinguishable】 if no efficient algorithm can tell the difference between them except with small probability.  
【问题】问题为<Σ，L，φ>，其中，Σ：问题所用的符号集合。Σ\*：Σ中有限串集合。其中L⊆Σ\*：问题的所有实例集合。φ：L→Σ\*。  
【判定问题】0，1∈Σ且φ（L)⊂{0，1},记为π=<Σ，L，φ>   
【P问题】DTM多项式时间可解问题。  
【NP问题】NTM多项式时间可解（DTM多项式时间可验证）问题。  
【NPC问题】若π∈NP，且对∀π’∈NP，都有π’∝π,则π∈NPC。  
【归约】将某个计算问题(https://zh.wikipedia.org/w/index.php?title=%E8%A8%88%E7%AE%97%E5%95%8F%E9%A1%8C&action=edit&redlink=1)转换为另一个问题的过程。  
【多项式变换】π1=<Σ1，L1，φ1>，π2=<Σ2，L2，φ2>之间，若存在以下变换，f:Σ1*→Σ2*的，f（L1）⊆L2，且∀I∈L1恒有φ1（I)=φ2（f（I）)。并且可用某个DTM程序在多项式时间内计算出f（I），则称π1可多项式变换为π2，记为π1∝π2。  
【数字签名】为保护消息产生者对消息的非否认性（即不可抵赖发出消息的行为）而使用私有的密钥或算法对消息进行完整性保护（生成签名串），该完整性保护的签名串其他人无法在多项式时间产生，且通常该完整性保护可以被其他人在多项式时间进行验证。  
【匿名性】同一主体在不同信息交换过程中所用身份的互相不可关联性。  
【形式化方法】在计算机科学领域特别是软件与硬件工程领域，对系统进行的基于数学的定义、开发、验证。是理论计算机科学的应用，包括逻辑演算、形式语言、自动机、离散事件动态系统、程序语义等。  
【区块链】把所有的交易信息（账目），组织成为一个不可篡改的链表，所形成的自带对账功能的数字记账技术。  
【访问控制】在主体和客体及资源之间的信息流动进行安全相关的控制。信息的读写流动称之为【访问】，发起访问的主动实体称之为【主体】，被访问的实体称之为【客体】  
【访问控制策略】Access control policies are high-level requirements that specify how access is managed and who may access information under what circumstances. For instance, policies may pertain to resource usage within or across organizational units or may be based on need-to-know, competence, authority, obligation, or conflict-of-interest factors. At a high level, access control policies are enforced through a mechanism that translates a user’s access request, often in terms of a structure that a system provides.(来自NIST的Access Control Policy and Implementation Guides)  
【强制访问控制策略】TCSEC最早给出了保护机密性的强制访问控制策略：以包含在客体中的信息敏感性和访问这些敏感性信息的主体的正式授权信息为基础，对访问进行限制的策略。如果推广到包括机密性的所有安全属性，那么强制访问控制策略可以定义为：系统通过某种预定的不受用户改变的规则对访问进行管理的策略。   
【安全标记】在客体上的一组可用于支持强制访问控制策略的安全属性信息项。  
【隐蔽信道/潜信道/潜通道=Covert Channel】隐蔽信道的概念是由Lampson最先提出，指不是被设计或本意不是用来传输信息的通信信道。在后续研究（强制访问控制策略）中，隐蔽信道主要用来指用来突破系统访问控制策略的不进行数据交换的通信信道，包括存储隐蔽信道和时间隐蔽信道。而Lampson定义中的合法的隐蔽信道被称之为阈下信道（subliminal channel），主要用于信息隐藏。在操作系统中隐蔽信道则具体定义为，进程以违反操作系统安全策略的方式传送信息的通信信道。在强制访问控制的机密性模型中隐蔽信道是一项严重威胁，因此安全操作系统标准TCSEC要求B2级系统要进行彻底的存储隐蔽信道分析，而B3级系统要同时进行时间隐蔽信道分析。但目前尚没有完全消除隐蔽信道的有效方法。  
【自主访问控制策略】在TCSEC中，自主访问控制策略（DAC）被定义为，根据主体（如用户、进程或 I/O 设备等）的身份和他所属的组限制对客体的访问。所谓的自主，是因为拥有访问权限的主体，可以直接（或间接）地将访问权限赋予其他主体（除非受到强制访问控制的限制）。  
【缓冲区溢出】指程序运行时，边界检查不严格的缓冲区被用户输入数据污染所导致的安全问题。溢出数据可通过覆盖返回地址、临时变量、指针等对程序控制流进行劫持，完成恶意行为。  
【软件安全缺陷】即安全漏洞，是软件在设计、实现、配置、运行时的安全弱点，可导致软件因非预期操作产生机密性、数据或系统完整性、可用性等安全属性的破坏。  
【模糊测试】通过向程序输入非预期的数据，并监控其输出或状态的异常，来发现程序中故障的方法。  
【程序的部分正确性】对于程序S，前断言P指明程序执行前，程序变量应有的性质（程序正确执行的前提），后断言Q刻画了程序结束时的程序状态空间（程序所应实现任务的描述）。如果P满足，且S执行终止，则终止时Q成立，那么称程序S是部分正确的。  
【程序的终止性】对于程序S，如果前断言P成立，则程序必终止，则称程序S是终止的。  
【程序的完全正确性】如果程序是部分正确的，且是终止的，则程序是完全正确的。  
【符号执行】程序分析的一种方法。使用符号变量代替具体值作为程序或函数的参数，并基于符号变量对程序进行模拟执行，获得程序的路径约束。通过对约束进行求解，可获得高测试覆盖率的测试用例集，从而更有效发现程序错误。  
【动态符号执行】以具体的数值作为输入执行程序代码，在程序实际执行路径的基础上，用符号执行技术进行分析，提取路径的约束表达式，根据路径搜索策略（深度、广度）对约束表达式进行变形，求解变形后的表达式并生成新的测试用例，不断迭代上面的过程，直到完成遍历程序的所有执行路径。

#### 简答  
1.信息系统通常由哪些部分组成？  
>答：信息系统通常由以下几部分组成：设备及环境、网络和通信设施、系统及应用软件、数据、信息系统的管理体系和对信息系统进行管理和使用的人员。  

2.对信息系统进行安全分级主要依据什么？  
>答：信息系统承载的使命、信息系统包含的价值、信息系统被破坏后造成的后果，在衡量其价值和后果时，不仅要考虑直接的价值和损失，还要考虑间接的价值和损失。  

3.信息系统的安全管理体系由哪些部分组成？  
>答：主要包括信息安全行政管理和信息安全技术管理两部分，前者包括信息安全策略及管理制度、信息安全机构及人员，后者包括信息系统安全建设和信息系统安全运维。  

4.简述GDPR.  
>答：GDPR=General Data Protection Regulation，即欧盟法规《通用数据保护条例》。法规是对关于个人数据保护和隐私保护的规范，适用于在欧洲开展业务的所有企业，要求这些企业处理个人数据的业务流程必须符合其对个人隐私和个人数据安全性保护的规定。法案于2018年5月开始强制实施，违反此规定的企业将面临公司全球收益4%或两千万欧元（取高者）的罚款。此法案对个人数据及个人隐私的保护起到了积极作用。  

5.简述STRIDE威胁建模。  
>答：  Spoofing、Tampering、Repudiation、Elevation of Privilege、DoS、Info Disclosure  

6.简述PDCA.  
>答：通过规划（Plan）、执行（Do）、查核（Check）、行动（Act）四阶段，确保每次的目标都能达成。PDCA循环就是按照这样的顺序进行质量管理，并且循环不止地进行下去的科学程序。  

7.Dolev-Yao威胁模型中，认为敌手能够做什么，不能够做什么？答：  
>敌手可以：  
1）能获得经过网络的任何消息；  
2）是网络的一个合法使用者，因而可以发起与任何其他用户的对话；  
3）有机会成为任何主体发出信息的接收者；  
4）可以冒充任何别的主体给任意主体发消息。  
>敌手不可以：  
1）不能猜到从足够大的空间中选出的随机数；  
2）没有正确的密钥，不能由给定的密文恢复明文；对于完善加密算法，也不能从给定的明文构造出正确的密文；  
3）不能求出私有部分，如私钥；  
4）不能控制计算环境中的私有区域。  
        
8.chmod u+s filename之后，filename文件的权限属性发生了什么变化？    
>答：为filename文件加上setuid标志，使(一般只拥有调用该文件的用户具有的权限的)filename文件在执行阶段具有文件所有者的权限。           

9.简述ruid和euid的区别。  
>答：ruid(real user ID)表示实际用户ID，永久不会变：当一个用户登录系统之后，该shell的ruid就是该用户了，     
从该根shell产生的所有进程都会继承ruid，即该用户；        
euid(effective user ID)表示有效用户ID，euid，用于系统决定用户对系统资源的访问权限，通常情况下等于ruid；      
若某可执行文件设置了suid位，当其被非owner用户执行时，该进程的euid是文件的owner(创建了该文件且设置了suid位)，ruid是执行者(登录执行者)。   

10.setuid位有什么用处？      
>答：对于一个可执行文件，在执行时，一般该文件只拥有调用该文件的用户具有的权限，而设置setuid位可使文件在执行阶段具有文件所有者的权限。

前断言逐步向后断言推导；  
循环：找一个断言，在执行循环时成立，在退出循环时仍成立（不变式）

其中，R表示通过通路的条件，r表示通过通路后y的变化。

10.分析以下代码，回答如何逆向得到input中的数据，不要求写出代码，写出完整思路即可。  

```c  
signed __int64 check()  
{  
  int v1; // [rsp+8h] [rbp-8h]  
  int i; // [rsp+Ch] [rbp-4h]  
  int j; // [rsp+Ch] [rbp-4h]  
  int k; // [rsp+Ch] [rbp-4h]  

  srand(0xDEADBEEF);  
  v1 = strlen(input);  
  for ( i = 0; i < v1; ++i )  
    input[i] += (char)random() % 64;  
  for ( j = 0; j < v1; ++j )  
    input[j] ^= random();  
  for ( k = 0; k < v1; ++k )  
  {  
    if ( input[k] != marg_num[k] )  
      return 0LL;  
  }  
  return 1LL;  
}  
```  
要点：随机数生成，逆向的顺序，异或。    
random() 函数在随机种子固定的情况下会产生固定的随机序列，所以可以写程序获取在种子是 0xDEADBEEF 时，random()产生的64个随机数。       
marg_num 可以通过IDA从二进制文件中获取到。逆向进行程序逻辑，先取随机序列的后32个与marg_num依次异或，然后将异或的结果依次减去         
随机序列前32个模64，就得到了input。即input[i] = (random_list[i+32] xor marg_num[i]) - (random_list[i] mod 64)。

【05】能不能设计一种不产生碰撞的哈希函数?为什么？      
哈希函数是根据任意长度的消息计算出固定长度的散列值，其中必然存在碰撞的情况。       
我们设想一个散列值为n比特的哈希函数，n比特可以容纳的散列值只有2^n个，因此当有消息为2^n+1个时，      
根据鸽巢原理，这2^n+1个消息中，必然存在两个消息产生的哈希值一样。
